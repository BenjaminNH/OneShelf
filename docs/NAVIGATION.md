# OneShelf 页面流程与导航结构

## 1. 文档目标

本文档用于在 PRD 与 MVP 基础上，定义 OneShelf 的页面结构、主导航方式与关键用户流程。

目标是回答以下问题：

- 用户打开应用后首先看到什么
- 用户如何从海报墙进入播放
- 搜索、评分、继续观看、媒体源管理分别放在哪里
- Android 手机场景下应采用什么导航结构

本文档优先服务于 MVP 与 Android 首发版本。

## 2. 导航设计原则

OneShelf 的导航必须围绕“手机端快速找片并播放”展开，而不是套用内容平台或文件管理器的通用结构。

### 2.1 核心原则

- 首页即媒体库，不额外设置独立聚合首页
- 用户打开应用后，应尽快看到海报墙
- 进入播放的路径要尽量短
- 评分、搜索、设置都属于辅助动作，不应抢占主导航层级
- Android 返回逻辑必须清晰，不能制造迷路感

### 2.2 推荐方案

推荐采用：



不推荐采用：

- 底部多 Tab 导航
- 抽屉式主导航
- “首页 / 发现 / 媒体库 / 我的”这类内容平台型结构

原因：

- OneShelf 的核心任务域单一，不值得拆成多个一级导航
- 搜索和设置是低频辅助功能，更适合作为动作入口
- 底部 Tab 会稀释媒体库作为核心主场景的地位

## 3. 顶层页面结构

MVP 阶段建议只有 5 个核心页面：

1. 媒体库首页
2. 搜索页
3. 详情页
4. 播放页
5. 设置页

其中只有“媒体库首页”是主入口页，其余均为从主入口衍生的任务页。

## 4. 页面职责定义

### 4.1 媒体库首页

这是应用默认启动页，也是整个产品最重要的页面。

#### 页面职责

- 展示海报墙主视图
- 在顶部一行展示最近观看
- 提供搜索入口
- 提供设置入口
- 提供媒体源异常、扫描状态等轻量提示

#### 页面结构建议

- 顶部留白或极轻量状态区
- 最近观看横向一行
- 海报墙网格
- 底部工具栏
- 空状态或错误状态区域

#### 页面动作

- 点击最近观看项：优先进入详情页，也可在后续版本评估是否直接续播
- 点击海报：进入详情页
- 点击搜索：进入搜索页
- 点击设置：进入设置页

### 4.2 搜索页

搜索页是功能页，不是一级导航页。

#### 页面职责

- 提供基础文件名搜索
- 展示实时或提交后的匹配结果
- 支持从结果进入详情页

#### MVP 限制

- 只搜索文件名
- 不在 MVP 做演员、番号、评分等复杂搜索

### 4.3 详情页

详情页承担“浏览媒体信息 + 触发播放”的双重职责。

#### 页面职责

- 展示封面、背景图、标题、简介、演员、番号等信息
- 展示本地评分
- 提供播放与继续播放入口
- 提供外部播放器入口

#### 页面动作

- 播放
- 继续播放
- 外部播放器打开
- 设置或修改本地评分

### 4.4 播放页

播放页是沉浸式任务页，应尽量减少干扰。

#### 页面职责

- 内置播放本地视频
- 提供最基础的播放控制
- 记录进度
- 适配横屏场景
- 在不替代显式控件的前提下，提供播放器快捷手势

#### 导航要求

- 从详情页进入
- Android 返回键优先退出播放页并返回详情页
- 不在播放页堆叠过多次级入口

#### 播放页交互要求

- 显式控件仍然是基础交互路径，不能只依赖手势
- 支持以下快捷手势：
  - 双击中间暂停/继续
  - 双击左侧后退
  - 双击右侧前进
  - 长按右侧临时倍速播放
- 前进/后退与长按倍速的默认值来自设置页，可由用户调整

### 4.5 设置页

设置页集中承载管理性质操作，不进入主浏览链路。

#### 页面职责

- 管理媒体源
- 手动重扫
- 查看权限状态
- 管理播放方式
- 管理播放器手势默认值
- 管理缓存与外部播放器偏好

## 5. App Shell 建议

### 5.1 不使用底部 Tab

MVP 不建议上底部导航栏。

原因：

- 一级任务只有一个，就是浏览媒体库
- 搜索是临时动作，适合单独页面或全屏搜索
- 设置属于管理页，不值得占一个长期可见的底部入口

### 5.2 使用底部工具栏

媒体库首页建议使用贴近底部安全区域的工具栏，而不是传统顶部 App Bar。

建议承载内容：

- 搜索入口
- 设置入口
- 后续可扩展的轻量筛选入口

设计原则：

- 工具栏应贴近拇指热区
- 搜索入口可以是类似 iOS 底部搜索框的显式控件，而不只是图标
- 设置按钮应与搜索入口并列，位于底部易触达区域
- 工具栏应避开系统手势冲突，尊重底部安全区

### 5.3 顶部区域策略

MVP 不建议在媒体库首页顶部放置高频操作按钮。

建议规则：

- 顶部可保留状态栏与少量留白
- 如需展示标题，应弱化处理，不占用主要交互区域
- 重要交互尽量下沉到屏幕下半部

### 5.4 最近观看区域

最近观看不是单独页面，而是媒体库首页顶部的一条横向内容带。

建议规则：

- 仅在存在播放记录时显示
- 数量不宜过多
- 一屏可快速扫读
- 不应挤占底部工具栏空间

## 6. 推荐路由结构

推荐使用如下路由层级：

```text
/
  -> LibraryHomePage

/search
  -> SearchPage

/detail/:mediaId
  -> DetailPage

/player/:mediaId
  -> PlayerPage

/settings
  -> SettingsPage

/settings/sources
  -> MediaSourcesPage
```

说明：

- 媒体库首页是根路由
- 搜索、详情、播放、设置都是从根路由压栈进入
- 设置页下的媒体源管理可以作为二级设置子页面

## 7. 关键用户流程

### 7.1 首次启动流程

```text
启动应用
  -> 媒体库首页空状态
  -> 点击添加媒体源
  -> 系统目录选择器
  -> 开始扫描
  -> 扫描完成
  -> 回到媒体库海报墙
```

设计要求：

- 首次进入时不要先展示空白海报墙
- 空状态需要直接给出“添加媒体源”主按钮
- 扫描过程应提供清晰但轻量的反馈

### 7.2 日常浏览与播放流程

```text
媒体库首页
  -> 点击海报
  -> 详情页
  -> 点击播放
  -> 播放页
  -> 返回
  -> 详情页
  -> 返回
  -> 媒体库首页
```

这是最重要的主流程，必须做到路径短、状态稳定、返回清晰。

### 7.3 继续观看流程

```text
媒体库首页
  -> 最近观看一行
  -> 点击某项
  -> 详情页
  -> 继续播放
  -> 播放页
```

说明：

- MVP 先走“最近观看 -> 详情页 -> 继续播放”的保守路径
- 后续版本如确认价值足够，可再评估“最近观看直接续播”

### 7.4 搜索流程

```text
媒体库首页
  -> 点击搜索
  -> 搜索页
  -> 输入文件名关键词
  -> 结果列表
  -> 点击结果
  -> 详情页
```

### 7.5 权限失效恢复流程

```text
媒体库首页或设置页
  -> 媒体源失效提示
  -> 点击重新授权
  -> 系统目录选择器
  -> 权限恢复
  -> 返回原页面
```

### 7.6 管理媒体源流程

```text
媒体库首页
  -> 设置
  -> 媒体源管理
  -> 添加 / 删除 / 重扫 / 重新授权
```

## 8. Android 交互规则

### 8.1 返回行为

- 在详情页按返回：回媒体库首页
- 在搜索页按返回：回媒体库首页，并保留媒体库当前滚动位置
- 在设置页按返回：回上一个页面
- 在播放页按返回：退出播放页回详情页

### 8.2 旋转行为

- 浏览页默认以竖屏为主
- 播放页允许并优先适配横屏
- 旋转不应导致当前媒体上下文丢失

### 8.3 系统选择器

- 添加媒体源与重新授权都应调用 Android 原生目录选择器
- 用户完成授权后应回到 OneShelf 内的原任务流

## 9. 页面优先级建议

### 9.1 P0 页面

- 媒体库首页
- 搜索页
- 详情页
- 播放页
- 设置页
- 媒体源管理页

### 9.2 P1 页面或子视图

- 按演员浏览页
- 按文件夹浏览页
- 扫描详情页
- 缓存管理页

## 10. 设计结论

MVP 阶段的最佳结构不是多入口，而是让媒体库成为唯一主入口。

最终建议如下：

- 首页就是媒体库海报墙
- 最近观看作为首页顶部一行
- 搜索和设置通过底部工具栏进入
- 详情页承担播放前的主要决策
- 播放页保持沉浸且返回路径清晰
- 设置页集中处理媒体源和管理操作

这套结构最符合 OneShelf 当前的产品定位：

- 本地优先
- 手机优先
- 海报墙优先
- 播放优先
