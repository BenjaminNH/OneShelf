# OneShelf 视觉方向与原型规范

## 1. 文档目标

本文档用于定义 OneShelf 的原型与视觉方向，作为后续所有线框图、中低保真原型、高保真视觉稿的统一依据。

这份文档的目标不是只描述审美倾向，而是让新的 agent、设计师或实现者在没有额外口头上下文的情况下，也能根据本文档稳定地产出符合 OneShelf 定位的界面。

适用范围：

- 原型图
- 低保真线框
- 中保真页面设计
- 高保真视觉方向
- Flutter UI 实现前的设计对齐

## 2. 设计背景

OneShelf 是一个：

- Android 优先
- 手机优先
- 本地优先
- 海报墙优先
- 播放优先
- 隐私优先

的本地媒体库应用。

它不是：

- 电视端媒体中心
- 内容平台型视频 App
- 泛文件管理器
- 营销型产品首页

因此 OneShelf 的视觉方向必须服务于以下事实：

- 用户主要在大屏手机上单手使用
- 用户打开应用后最想做的是浏览海报墙和快速播放
- 用户已有本地元数据与封面，不需要界面抢戏
- UI 应该增强“私人片库”和“本地收藏柜”的感觉，而不是喧宾夺主

## 3. 最终选定方向

### 3.1 主方向名称

`Frosted Archive`

### 3.2 一句话定义

> 一个偏冷静、克制、私密的本地媒体库界面：让海报成为主角，让操作下沉到底部，让玻璃感只作为触控层次而不是全屏特效。

### 3.3 方向关键词

- Frosted
- Archive
- Quiet
- Private
- Cinematic
- Thumb-first
- Dense but calm

## 4. 为什么选这个方向

`Frosted Archive` 是以下两种倾向的融合：

- `Moon Glass`：提供高级、轻微未来感、底部悬浮工具区
- `Archive Mono`：提供整理感、秩序感、工具属性

最终目的：

- 保留 OneShelf 作为“私人媒体库”的精致感
- 避免把界面做成天气 App、营销页或纯播放器炫技页
- 让 UI 服务内容，而不是盖过内容

## 5. 明确不采用的方向

### 5.1 不做营销页风格

不采用 `$ui-ux-pro-max` 默认输出中的下列倾向：

- Video-first hero
- 大面积 Hero Banner
- 紫色 SaaS 配色
- Inter + 营销页型极大标题

原因：

- 这些更像 landing page，不像本地媒体库应用

### 5.2 不做整屏高透明玻璃

不允许：

- 整个页面铺满强毛玻璃
- 所有卡片都使用重透明
- 让背景纹理干扰海报辨识度

原因：

- 会显得廉价且影响可读性

### 5.3 不做高饱和霓虹娱乐风

不允许：

- 大面积蓝紫霓虹
- 强发光边框
- 夜店式高对比荧光配色

原因：

- 会削弱本地收藏工具的长期使用舒适度

## 6. 视觉性格定义

### 6.1 气质

OneShelf 应该呈现为：

- 安静
- 精致
- 稍带科技感
- 但不炫耀技术感

用户看到界面时的感受应该是：

- 这是我的私人收藏柜
- 这是一个适合夜间和单手使用的工具
- 它有质感，但不过度表演

### 6.2 情绪温度

- 主体是偏冷静的深灰蓝
- 局部点缀采用暖琥珀，用于评分、进度、当前选中状态
- 不要让整体页面偏冷到无生命，也不要偏暖到像流媒体娱乐平台

## 7. 色彩方向

以下不是最终像素级设计稿，而是强约束的色彩区间。

### 7.1 主背景

- 深石墨蓝灰
- 深雾蓝黑
- 偏低饱和、低亮度

推荐范围：

- `#0F1318`
- `#131922`
- `#1A2230`

### 7.2 次级表面

用于卡片、详情区、底部工具栏的玻璃或磨砂面：

- `rgba(180, 195, 215, 0.12)`
- `rgba(200, 215, 235, 0.16)`
- 带轻微冷色边缘高光

### 7.3 文本颜色

- 主文字：偏暖白而不是纯白
- 次文字：低对比灰蓝

推荐范围：

- 主文字：`#F3F5F7`
- 次文字：`#B6BEC9`
- 弱文字：`#8D98A6`

### 7.4 强调色

只允许一个主要强调色，用于：

- 本地评分
- 播放进度
- 当前选中状态
- 少量关键操作

推荐方向：

- 暖琥珀
- 蜂蜜金

推荐范围：

- `#D9A441`
- `#E0B35A`
- `#F2A23A`

### 7.5 禁止色彩

不要作为主风格使用：

- 高饱和紫
- 亮蓝霓虹
- 纯黑 + 纯白的极端撞色
- 糖果色多色并存

## 8. 材质与层次

### 8.1 玻璃感使用原则

玻璃感只能出现在这些地方：

- 底部悬浮工具栏
- 部分浮层
- 少量次级控制面板

不应出现在：

- 海报墙底板
- 主体背景整页
- 所有卡片无差别统一套用

### 8.2 阴影与边框

阴影应轻而软，目的是把悬浮层从内容区抬起来。

建议：

- 外阴影轻
- 内高光弱
- 细边框半透明

不建议：

- 厚重投影
- 强发光外轮廓
- 硬边金属质感

### 8.3 圆角

OneShelf 整体圆角应该统一，避免组件语言混乱。

建议：

- 海报卡片：中等圆角
- 底部胶囊工具栏：大圆角
- 圆形按钮：完整圆形

整体观感应偏柔和，而不是尖锐工业风。

## 9. 排版方向

### 9.1 总体原则

- 不使用营销页式超大标题
- 不让标题系统抢海报的视觉主导权
- 文字层级必须清楚但克制

### 9.2 字体性格

若后续进入高保真或实现阶段，建议选择：

- 一个中性但有高级感的正文字体
- 一个略带识别度但不夸张的标题字体

字体应满足：

- 长时间阅读舒适
- 数字和番号清晰
- 中文显示稳定

### 9.3 文字布局

- 页标题可以弱化
- 详情页标题可适度突出
- 元数据标签应更像资料信息，不像营销文案

## 10. 页面结构总原则

### 10.1 顶部区域

顶部区域应保持极轻量。

允许：

- 状态栏
- 少量留白
- 轻标题或媒体源状态提示

不允许：

- 顶部 AppBar 上堆高频图标
- 把搜索和设置放在顶部角落
- 让主交互发生在屏幕最上缘

### 10.2 底部区域

底部区域是 OneShelf 的高频操作核心。

必须满足：

- 靠近底部安全区
- 单手可达
- 清晰可点
- 视觉上独立于海报墙

### 10.3 内容区

内容区应突出海报，不要过多框线和装饰。

推荐：

- 海报墙高密度但不拥挤
- 最近观看只占一行
- 大部分空间留给媒体内容本身

## 11. 核心组件规范

### 11.1 底部悬浮工具栏

这是 OneShelf 的核心识别锚点，也是后续新 agent 画图时必须保留的设计母题。

#### 结构

- 一个贴近底部安全区的悬浮胶囊条
- 中间为显式搜索入口
- 右侧为设置圆形按钮
- 左侧可预留轻量入口，但 MVP 可为空或后置

#### 视觉特征

- 半透明磨砂
- 轻阴影
- 冷色玻璃表面
- 不要太厚重

#### 交互特征

- 搜索入口必须显式、易点击
- 设置按钮必须在拇指热区
- 工具栏不能遮挡核心海报内容

#### 禁止事项

- 不做传统底部 tab bar
- 不做 4 到 5 个平均分布图标
- 不做顶部搜索按钮 + 底部再放操作的重复结构

### 11.2 最近观看条

#### 结构

- 位于内容区顶部
- 横向滑动或横向排列
- 只显示少量条目

#### 视觉要求

- 比主海报墙更轻
- 更像“最近使用”提示，而不是第二个主列表

### 11.3 海报卡片

#### 结构

- 海报图为绝对主视觉
- 底部可有轻量信息，如标题、评分、观看进度

#### 要求

- 优先展示图片
- 文字不应挤占海报面积
- 缺图时也必须保持网格稳定

### 11.4 评分控件

#### 形式

- 详情页内呈现
- 使用 0-10 的显式分值
- 可通过底部弹层、滑杆或分段点击方式设置

#### 视觉要求

- 强调色使用暖琥珀
- 不要做成流媒体平台那种巨大评分徽章

## 12. 页面级原型要求

下面的要求是给后续画原型的 agent 的直接指令。

### 12.1 媒体库首页

必须包含：

- 顶部轻量留白区
- 一行最近观看
- 海报墙主视图
- 底部悬浮胶囊工具栏

必须避免：

- 顶部 AppBar 图标
- 大标题英雄区
- 多 tab 一级导航

画图重点：

- 海报墙密度
- 底栏与内容的空间关系
- 单手可操作性

### 12.2 搜索页

必须包含：

- 可聚焦的搜索输入区
- 文件名结果列表或海报结果
- 空结果提示

必须避免：

- 复杂筛选器
- 演员、番号等进阶搜索入口

画图重点：

- 搜索从底部入口触发后的过渡关系
- 搜索结果是否保持媒体视觉语言一致

### 12.3 详情页

必须包含：

- 大封面或大封面主体
- 背景图或模糊背景层
- 播放按钮
- 继续播放入口
- 外部播放器入口
- 本地评分
- 基本元数据

必须避免：

- 信息过多挤满首屏
- 顶部堆一排图标按钮

画图重点：

- 播放操作区落点
- 评分与元数据的层级
- 内容氛围和工具感的平衡

### 12.4 播放页

必须包含：

- 简洁控制条
- 清晰返回逻辑
- 横屏主用场景

必须避免：

- 复杂菜单
- 与播放无关的视觉干扰

### 12.5 设置页

必须包含：

- 媒体源管理入口
- 扫描入口
- 播放与缓存相关设置

必须避免：

- 像系统设置页那样过度密集
- 把设置做成产品的视觉主角

## 13. 原型绘制优先级

后续新 agent 若开始出图，应按以下顺序绘制：

1. 媒体库首页
2. 详情页
3. 搜索页
4. 播放页
5. 设置页

原因：

- 媒体库首页定义了整体视觉母体
- 详情页定义了信息层级与播放动作
- 搜索页定义了底部入口如何展开
- 播放页与设置页都依附前两者语言

## 14. 给后续新 Agent 的执行指令

如果你是后续接手 OneShelf 原型绘制的新 agent，请遵循以下硬规则：

1. 先阅读 `PRD.md`、`MVP.md`、`NAVIGATION.md`、`DATA_MODEL.md` 与本文档。
2. 把 `Frosted Archive` 视为当前唯一主视觉方向。
3. 首页必须是媒体库海报墙，而不是独立营销首页。
4. 高频操作必须下沉到底部，不得回退为传统顶部 AppBar 主操作。
5. 底部悬浮胶囊工具栏必须成为视觉锚点。
6. 玻璃感只能点到为止，不允许全屏毛玻璃泛滥。
7. 海报内容优先，UI 只能服务内容，不能压过内容。
8. 评分、继续观看、搜索都要保留，但必须是克制而实用的表达。

## 15. 设计决策记录

### 决策 1：选定 `Frosted Archive` 作为主方向

- 原因：最能平衡私人媒体库气质、手机单手操作和可实现性

### 决策 2：底部悬浮工具栏作为核心识别元素

- 原因：符合大屏手机拇指热区使用习惯，也与参考案例一致

### 决策 3：顶部区域极轻量

- 原因：避免把高频操作放在难触达区域，让内容成为主角

### 决策 4：玻璃感只作为局部层次

- 原因：保持质感，同时避免廉价和可读性问题

### 决策 5：不采用营销页式视觉系统

- 原因：OneShelf 是工具型本地媒体库，不是品牌展示页

## 16. 参考说明

本方向同时参考了：

- 当前已确认的 OneShelf 产品定位与导航结构
- 大屏手机单手使用场景
- 底部悬浮操作区的交互思路
- Apple 对 toolbar / layout 的稳定设计原则

但应注意：

- OneShelf 不是 iOS 系统应用复刻
- 不直接照搬天气 App
- 重点是借鉴“下沉操作、易触达、内容优先”的原则
