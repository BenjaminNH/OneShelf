# OneShelf 竞品调研

> 调研日期：2026-04-02

## 一、竞品总览

### 直接竞品：无

市场上不存在同时满足以下条件的应用：

- Flutter 开发 + 手机竖屏优先
- 纯本地 NFO/poster/fanart 读取（不依赖在线刮削）
- 内置电影播放器 + 海报墙浏览
- 隐私优先 / 离线可用

### 间接竞品

| 应用 | Stars | 语言 | 定位 | 与 OneShelf 关系 |
|------|-------|------|------|-----------------|
| **Nova Video Player** | 4.4k | Java/C++ | 开源视频播放器，全平台 | 功能最接近，但 TV 优先 + 依赖 TMDB 在线刮削，手机触屏体验差 |
| **Namida** | 5.2k | Flutter | 音乐播放器 + MV 播放 | 同技术栈，但视频模块仅围绕 MV 设计，无电影海报墙/NFO 读取/进度记录 |
| **ShowCase / MovieDB** | 301 | Kotlin | 电影收藏追踪管理 | 不播放视频、不读本地文件，依赖 TMDB/Trakt 在线 API |
| **Kodi** | — | C++/Python | 全功能媒体中心 | PRD 已提到，TV 优先，手机体验差，过于臃肿 |
| **LitPlayer** | 67 | Kotlin (Compose) | 本地视频播放器 | 更偏纯播放器，无海报墙/NFO 元数据读取 |

## 二、核心竞品详细对比

### Nova Video Player (`nova-video-player/aos-AVP`)

- **优势**: 功能最全面的开源本地播放器，支持 SMB/WebDAV/NAS，海报墙成熟
- **劣势**: 
  - 依赖 TMDB/Trakt 在线刮削，离线能力弱
  - UI 偏电视风格，手机竖屏体验差
  - 体量大而全，不够轻量
- **威胁等级**: ⭐⭐⭐⭐ — 最接近的开源竞品，但手机端体验是 OneShelf 的差异化机会

### Namida (`namidaco/namida`)

- **优势**: Flutter 构建，UI 精美，本地文件扫描能力强
- **劣势**: 
  - 核心是音乐播放器，视频仅限 MV 场景
  - 依赖网络（YouTube 流媒体、在线封面）
  - 不读 NFO 元数据
  - EULA 许可证，非开源，不可 fork
- **威胁等级**: ⭐ — 产品定位完全不同，不构成竞争；仅可作为 Flutter 视频播放的技术参考

## 四、结论

**OneShelf 的 niche 目前无人覆盖。**

核心差异化：
1. 唯一专注「已整理好的本地媒体目录」的手机端应用 — 不刮削、不编辑、不云端
2. 精准兼容 JavSP 等工具生成的目录结构（NFO + poster.jpg + fanart.jpg）
3. Flutter 技术栈，V2 具备跨平台扩展潜力
4. 手机竖屏浏览 + 横屏播放的产品设计，填补 Nova 等 TV 优先应用的空白

**建议参考对象**：
- 功能设计参考：Nova Video Player
- Flutter 技术参考：Namida（视频播放模块）
